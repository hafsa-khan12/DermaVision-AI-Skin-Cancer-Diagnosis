How to Use DermaVision AI

This guide explains how to upload a skin lesion image, run AI diagnosis, interpret confidence score, understand risk levels, and generate a professional report.

5

Simple Steps

Quick Start Guide

1

Open Scans Page

Go to the Scans page from the sidebar or click New Scan from the top bar.

2

Upload Image

Select a JPG, JPEG, or PNG skin lesion image from your computer.

3

Run Diagnosis

Click Run Diagnosis and wait for the AI model to generate prediction results.

4

Review Result

Check predicted class, confidence score, risk level, and AI-generated summary.

5

Download Report

Click Download PDF Report and choose Save as PDF from the print dialog.

Risk Level Guide

Low Risk

Usually shown when the model predicts benign class with strong confidence. Routine monitoring is suggested.

Medium Risk

Used when prediction is uncertain or confidence is moderate. Clinical confirmation is recommended.

High Risk

Used when the model predicts malignant class with high confidence. Dermatologist consultation is strongly recommended.

Supported Image Rules

File Types JPG, JPEG, PNG
Recommended Image Clear lesion image with good lighting
Image Size Automatically resized to 224 × 224
Input Type Skin lesion / dermoscopic image
Output Prediction, confidence, risk level, summary

Common Issues

Why is the report button not working?

First run diagnosis by uploading an image. Report generation works only after prediction result is available.

Why does camera capture not open?

Allow camera permission in the browser. If permission is blocked, upload an image manually.

Why does Internal Server Error appear?

Usually it means a template file is missing, file name is misspelled, or HTML/Jinja syntax has an error.

Why is confidence not equal to medical certainty?

Confidence is only the model probability score. It does not replace clinical diagnosis.

Troubleshooting Checklist

  • Make sure server is running with python -m uvicorn app:app --reload
  • Check that templates folder contains all required HTML files.
  • Make sure file name is exactly scans.html, not scan.html or scans.htnl.
  • Open /health to verify model is loaded.
  • Use Ctrl + F5 to hard refresh browser after CSS/HTML changes.
  • Check terminal traceback if a page shows Internal Server Error.
